使用 React 定义 UI 结构：使用 JSX 语法编写一个组件，结构由 React 核心库创建并管理。

Babel 实时编译：由于浏览器无法直接识别 JSX，babel-standalone 会拦截带有 type="text/jsx" 属性的 <script> 标签，并实时地将 JSX 代码编译成标准的 JavaScript。

ReactDOM 渲染到页面：编译完成后，ReactDOM.render(...) 方法被调用，将 React 组件生成最终的 HTML 元素并真实地“画”在网页的指定容器内。

Day 3 & 4

实现插件的方式可选：IDE 插件， CLI 工具， Web 服务

 IDE 插件
特点
深度集成：可以访问当前打开的文件、选中的代码、编辑器状态，甚至利用 IDE 的诊断框架（如 VSCode 的 Diagnostic）。

交互性强：可以通过按钮、菜单、悬浮提示、侧边栏树形视图等提供直观操作。

适合场景：无缝体验，不弹出终端窗口。

CLI 工具
特点
无依赖：只需要一个可执行文件（或 Python 脚本），不依赖图形界面。

可脚本化：支持参数、配置文件、环境变量。

实现难度简单

输入输出简单：读取文件或标准输入，输出到标准输出或文件。

无需 UI：不需要处理鼠标事件、窗口布局。

打包简单：可以发布为 PyPI 包（pip install）或编译为单个二进制。

测试容易：单元测试即可覆盖大部分逻辑。

Web 服务
特点
跨平台零安装：用户只需打开浏览器，无需安装任何软件。

易于协同：多人可以同时访问同一服务，共享报告。

持久化：可将数据存入数据库，长期跟踪历史版本。

实现难度高
